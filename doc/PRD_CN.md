# AI 足球量化终端 — 产品需求文档 (PRD)

> 版本: 1.1
> 日期: 2026-02-27
> 状态: 开发中

---

## 1. 产品概述

### 1.1 愿景
一个彭博终端风格的实时足球分析平台，在直播比赛期间展示 AI 预测概率、量化指标和市场 Edge 分析。面向 YouTube/TikTok 直播，采用会员分层变现模式。

### 1.2 产品定位
- **是什么**：足球比赛的实时数据可视化终端
- **不是什么**：博彩平台、比赛转播工具、视频回放系统
- **合规性**：仅数据可视化 — 不含比赛画面、音频或逐回合复刻
- **数据延迟**：距离真实事件最低延迟 30 秒（合规要求）

### 1.3 目标用户
| 用户类型 | 核心需求 | 对应等级 |
|----------|----------|----------|
| 普通球迷 | 实时比分 + 基础概率 | Free |
| 足球数据爱好者 | 市场 Edge、因素解释、量化指标 | Pro |
| 专业玩家 / 数据科学家 | 不确定性分析、校准指标、CI 区间、报告导出 | Elite |
| 内容创作者 / 直播主 | 完整终端画面用于直播 | Pro/Elite |

## 2. 核心功能

### 2.1 实时比赛终端（Free）

**5 层 UI 架构：**

```
第 1 层：状态栏
  联赛 | 轮次 | 主队 比分 客队 | 分钟 | 半场
  健康状态 | 数据延迟 | 数据来源

第 2 层：概率区
  1X2 AI 概率（主胜 / 平局 / 客胜）
  Delta 指示器（相对上次更新的变化）
  模型置信度

第 3 层：趋势区
  Sparkline 折线图：概率随时间变化
  压力指数趋势
  xG 差值趋势

第 4 层：统计区
  双列球队对比
  射门、控球率、xG、角球、犯规、红黄牌
  事件流（Free 限 3 条）

第 5 层：量化区
  压力指数、动量分数
  波动率指数、失球风险
  预期进球时间窗口、模型方差
```

### 2.2 市场 Edge（Pro）
- **市场隐含概率**：从真实赔率反推
- **Edge（AI - 市场）**：AI 预测与市场共识的差值
- **真实赔率展示**：来自 The Odds API 的实时赔率
- **变现逻辑**：用户升级 Pro 的核心驱动力

### 2.3 解释面板 — Why Δ（Pro）
- 驱动概率变化的 Top 3 因素
- 可视化影响条（含上升/下降方向）
- 因素包括：射正次数、压力指数、进球、红牌、控球率变化、xG 差值、时间衰减
- 建立用户对模型的信任

### 2.4 事件流（Free: 3 条，Pro: 5-8 条）
- 彭博风格事件日志
- 支持事件类型：GOAL、YELLOW、RED、SUB、VAR、PENALTY
- 最新在前，含分钟标记
- 按事件类型颜色编码

### 2.5 不确定性与校准（Elite）
- CI95 置信区间（每个结果）
- 滚动 Brier Score（20 分钟窗口）
- 锐度指标（基于信息熵）
- 蒙特卡洛模拟次数
- 专业级模型质量证据

### 2.6 管理后台
- 基于 Web 的比赛管理面板（/admin/）
- 增删改查管理比赛
- 健康状态仪表盘
- 比赛配置：球队名、Elo 评分、API ID、赔率类型

## 4. 技术架构

### 4.1 系统组件
```
[前端: React 18 + Vite]
    |
    |-- WebSocket (ws://host/ws/{match_id})
    |-- REST API (http://host/api/*)
    |
[Nginx 反向代理]
    |
    |-- / → 前端 (SPA)
    |-- /api/ → 后端
    |-- /ws/ → 后端 (WebSocket 升级)
    |-- /admin/ → 后端 (静态文件)
    |
[后端: FastAPI Python]
    |
    |-- WebSocket 管理器（单场广播）
    |-- LivePredictor（XGBoost + 泊松修正）
    |-- 市场引擎（隐含概率 + Edge）
    |-- 解释引擎（Delta 关键因素）
    |-- 不确定性引擎（CI95、Brier、锐度）
    |-- 比赛状态存储（单场 Delta、事件、滚动指标）
    |-- 数据客户端（AllSportsApi、SportMonks、API-Football、The Odds API）
    |
[Redis]
    |-- 缓存层
    |-- 未来：PubSub 广播、会话存储
```

### 4.2 数据流
```
外部 API（AllSportsApi / SportMonks / Odds API）
    ↓（30 秒轮询）
后端数据客户端
    ↓
特征构建 → LivePredictor（XGBoost）
    ↓
比赛状态存储（Delta 计算、事件去重）
    ↓
引擎流水线：市场 → 解释 → 不确定性 → 量化
    ↓
v1.1 JSON 载荷组装
    ↓（每 2 秒 WebSocket 推送）
前端 mapPayload.js → React 组件
    ↓
渲染终端 UI（1920x1080）
    ↓（OBS Browser Source）
YouTube / TikTok 直播推流
```

### 4.3 WebSocket 协议
- **地址**: `ws://host/ws/{match_id}`
- **推送间隔**: 2 秒
- **重连策略**: 斐波那契退避（3s → 5s → 8s → 13s）
- **载荷格式**: v1.1 JSON Schema（10 个顶级区块）
- **健康状态**: OK（绿）、DEGRADED（黄）、STALE（红）
- **序列号**: 单调递增的 `meta.seq` 用于排序

### 4.4 部署方案
- **容器化**: Docker Compose（4 个服务）
- **服务**: Redis 7、后端（Python 3.11）、前端（Node 20 构建 + nginx）、Nginx（反代）
- **端口**: 80（nginx）、8000（后端）、5173（前端直连）、6379（redis）

## 5. AI/ML 模型

### 5.1 赛前模型
- **算法**: 校准 XGBoost 分类器
- **特征（13 个）**: 赔率隐含概率（3）、Elo 评分（2）、射门统计（4）、xG 统计（2）、角球（2）
- **分类**: 主胜 / 平局 / 客胜
- **输出**: 概率向量 + 置信度分数
- **训练数据**: 3000 场比赛（当前为模拟数据，需升级为真实数据）

### 5.2 赛中修正
- **方法**: 基于泊松分布的实时调整
- **输入**: 当前比分、比赛分钟数、剩余时间
- **行为**: 领先方概率随时间衰减递增；平局概率在比赛后期收缩

### 5.3 评估指标
| 指标 | 当前值 | 目标值 |
|------|--------|--------|
| 交叉验证准确率 | ~47%（模拟数据） | >50%（真实数据） |
| Brier Score | 尚未追踪 | <0.22 |
| LogLoss | 尚未追踪 | <0.95 |
| 校准度 | 尚未验证 | 5% 区间内 |

## 6. 会员分层

### 6.1 免费版（Free）
- 1X2 AI 概率 + Delta 变化
- 趋势 Sparkline（概率、压力、xG）
- 基础统计（射门、控球率、xG）
- 事件流（最近 3 条）
- 30 秒数据延迟

### 6.2 专业版 Pro（$9.99/月）
- 包含 Free 全部内容，加上：
- **市场隐含概率 + Edge（AI vs 市场）** — 核心变现功能
- **解释面板（Why Δ）** — Top 3 因素
- 完整事件流（5-8 条）
- 完整量化指标（压力指数、波动率、失球风险、进球窗口）
- 更快数据刷新（目标：15 秒延迟）

### 6.3 精英版 Elite（$29.99/月）
- 包含 Pro 全部内容，加上：
- **不确定性分析**：CI95 区间、滚动 Brier、锐度、MC 模拟次数
- 多场比赛并行终端（3-4 场同时）
- 半场/全场报告导出（JSON + 图片快照）
- 历史关键时刻回放
- API 接口（未来）

## 7. 合规与法律

### 7.1 版权安全
- 页面展示 "DATA VISUALIZATION ONLY" 水印
- 不使用联赛官方 Logo
- 不播放比赛画面/音频/逐回合进攻
- 最低 30 秒数据延迟
- 所有数据来自授权 API 提供方

### 7.2 博彩合规
- 平台仅提供分析/可视化
- 不直接集成投注功能
- 不做任何理财建议声明
- 免责声明："仅供教育与娱乐用途"
